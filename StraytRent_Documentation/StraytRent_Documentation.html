📋 1. Project Overview

1.1 What is StraytRent?

StraytRent is a verified rental marketplace connecting UNILAG students with trusted landlords, caretakers, and agents. The platform eliminates agent exploitation, provides secure escrow payments, and ensures accurate property listings through a unique freshness verification system.

🎯 Core Value Proposition

  • For Students: Verified listings, transparent pricing, secure payments, and dispute resolution
  • For Landlords: Verified tenants, guaranteed payments, reduced vacancy periods
  • For Agents: Volume-based commissions, verified leads, permanent higher tier for founding partners

1.2 Problem Statement

Metric Value Impact
UNILAG Students 45,000+ Massive demand for off-campus housing
On-campus beds ~8,000 Only 18% can live on campus
Students off-campus 37,000+ Forced into expensive private rentals
Average rent (self-con) N1.2M - N1.7M/year Unaffordable for many students
Agent fees 10-30% Added on top of inflated rent

1.3 Technology Stack

Layer Technology Rationale
Backend Node.js + Express Fast, scalable, excellent for API development
Database PostgreSQL via Supabase ACID compliance, real-time subscriptions, built-in auth
Authentication Email OTP Passwordless, secure, user-friendly
Email Service Resend / SendGrid / Nodemailer Reliable email delivery for OTPs and notifications
Payments Paystack Nigerian-native, escrow support, split payments

2.2 Database Schema Overview

The database consists of 15 core tables organized around the rental lifecycle:

Table Name Purpose Key Relationships
profiles User extended information (role, KYC, ratings) Links to auth.users
otp_codes Email OTP storage and verification Stores temporary auth codes
listings Property/unit listings Belongs to landlord, managed by caretaker
inspections Property viewing appointments Links students to listings
reservations 72-hour unit holds Converts to tenancy agreement
tenancy_agreements Digital lease contracts Core transaction record
escrow_transactions Payment holding and release Tied to tenancy agreements
handover_checklists Move-in/move-out verification Required for escrow release

3.1 Profiles Table

CREATE TABLE public.profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  role TEXT NOT NULL CHECK (role IN ('student', 'landlord', 'caretaker', 'agent')),
  full_name TEXT NOT NULL,
  email TEXT UNIQUE NOT NULL,
  phone_number TEXT,
  school_id_url TEXT,
  government_id_url TEXT,
  bvn TEXT,
  selfie_url TEXT,
  kyc_verified BOOLEAN DEFAULT FALSE,
  verified_badge BOOLEAN DEFAULT FALSE,
  average_rating DECIMAL(3,2) DEFAULT 0,
  total_ratings INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

3.2 OTP Codes Table (Email Authentication)

CREATE TABLE public.otp_codes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT NOT NULL,
  code TEXT NOT NULL,
  purpose TEXT DEFAULT 'login',
  expires_at TIMESTAMPTZ NOT NULL,
  used BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Indexes for performance
CREATE INDEX idx_otp_email ON public.otp_codes(email);
CREATE INDEX idx_otp_code ON public.otp_codes(code);
CREATE INDEX idx_otp_expires ON public.otp_codes(expires_at);

3.3 Listings Table

CREATE TABLE public.listings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  landlord_id UUID REFERENCES profiles(id) NOT NULL,
  caretaker_id UUID REFERENCES profiles(id),
  agent_id UUID REFERENCES profiles(id),
  title TEXT NOT NULL,
  description TEXT,
  unit_type TEXT CHECK (unit_type IN ('self-con', 'mini-flat', 'hostel-bed', 'shared-room')),
  address TEXT NOT NULL,
  walk_time_to_gate_minutes INTEGER,
  area TEXT CHECK (area IN ('Onike', 'Abule Oja', 'Akoka', 'Iwaya', 'Bariga', 'Sabo')),
  annual_rent NUMERIC(12,2) NOT NULL,
  platform_fee_percent NUMERIC(5,2) DEFAULT 6.0,
  security_deposit NUMERIC(12,2),
  utilities JSONB DEFAULT '{}',
  rules JSONB DEFAULT '{}',
  status TEXT DEFAULT 'pending_verification',
  last_confirmed_at TIMESTAMPTZ,
  needs_confirmation BOOLEAN DEFAULT TRUE,
  is_verified BOOLEAN DEFAULT FALSE,
  view_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

🔐 5. Email OTP Authentication

5.1 How Email OTP Works

📧 Passwordless Authentication Flow

  1. User enters email on login/register form
  2. Backend generates 6-digit OTP and stores in otp_codes table (10-minute expiry)
  3. Email sent via Resend/SendGrid/Nodemailer with OTP code
  4. User enters OTP in the application
  5. Backend verifies OTP (checks expiry, unused status)
  6. If new user: Creates auth user and profile
  7. If existing user: Returns JWT token for authenticated sessions
  8. Welcome email sent to new users based on their role

5.3 Security Features

  • 10-minute OTP expiry - Prevents replay attacks
  • One-time use only - OTP marked used after verification
  • Rate limiting - 5 OTP requests per email per hour
  • No password storage - Eliminates password breach risks
  • JWT tokens - Stateless session management
  • Automatic OTP cleanup - Cron job removes expired codes hourly
✅ Best Practice: In production, always use HTTPS and implement rate limiting on OTP endpoints to prevent abuse.

6.4 Complete End-to-End Test Script

Save as test-flow.sh and run with bash test-flow.sh:

#!/bin/bash

BASE_URL="http://localhost:3000"
STUDENT_EMAIL="student@test.com"
LANDLORD_EMAIL="landlord@test.com"

echo "=== StraytRent Complete Flow Test ==="

# 1. Student registration
echo "1. Sending OTP to student..."
curl -s -X POST $BASE_URL/api/auth/send-otp \
  -H "Content-Type: application/json" \
  -d "{\"email\": \"$STUDENT_EMAIL\", \"purpose\": \"login\"}"

echo "\n2. Verifying student OTP..."
STUDENT_RESPONSE=$(curl -s -X POST $BASE_URL/api/auth/verify-otp \
  -H "Content-Type: application/json" \
  -d "{\"email\": \"$STUDENT_EMAIL\", \"code\": \"123456\", \"full_name\": \"John Student\", \"role\": \"student\"}")
STUDENT_TOKEN=$(echo $STUDENT_RESPONSE | jq -r '.session.access_token')
echo "Student token obtained"

# 2. Landlord registration
echo "3. Sending OTP to landlord..."
curl -s -X POST $BASE_URL/api/auth/send-otp \
  -H "Content-Type: application/json" \
  -d "{\"email\": \"$LANDLORD_EMAIL\", \"purpose\": \"login\"}"

echo "\n4. Verifying landlord OTP..."
LANDLORD_RESPONSE=$(curl -s -X POST $BASE_URL/api/auth/verify-otp \
  -H "Content-Type: application/json" \
  -d "{\"email\": \"$LANDLORD_EMAIL\", \"code\": \"123456\", \"full_name\": \"Jane Landlord\", \"role\": \"landlord\"}")
LANDLORD_TOKEN=$(echo $LANDLORD_RESPONSE | jq -r '.session.access_token')
echo "Landlord token obtained"

# 3. Create listing
echo "5. Creating listing..."
LISTING_RESPONSE=$(curl -s -X POST $BASE_URL/api/listings \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LANDLORD_TOKEN" \
  -d '{
    "title": "Test Property",
    "description": "Test description",
    "unit_type": "self-con",
    "address": "Test Address",
    "walk_time_to_gate_minutes": 5,
    "area": "Onike",
    "annual_rent": 900000
  }')
LISTING_ID=$(echo $LISTING_RESPONSE | jq -r '.listing.id')
echo "Listing created: $LISTING_ID"

# 4. Book inspection
echo "6. Booking inspection..."
INSPECTION_RESPONSE=$(curl -s -X POST $BASE_URL/api/inspections \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $STUDENT_TOKEN" \
  -d "{
    \"listing_id\": \"$LISTING_ID\",
    \"scheduled_at\": \"2026-05-01T14:00:00Z\"
  }")
INSPECTION_ID=$(echo $INSPECTION_RESPONSE | jq -r '.inspection.id')
echo "Inspection booked: $INSPECTION_ID"

# 5. Create reservation
echo "7. Creating reservation..."
RESERVATION_RESPONSE=$(curl -s -X POST $BASE_URL/api/reservations \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $STUDENT_TOKEN" \
  -d "{
    \"listing_id\": \"$LISTING_ID\",
    \"inspection_id\": \"$INSPECTION_ID\"
  }")
RESERVATION_ID=$(echo $RESERVATION_RESPONSE | jq -r '.reservation.id')
echo "Reservation created: $RESERVATION_ID"

echo "=== Test Complete ==="

🔧 8. Troubleshooting Guide

8.1 Common Errors and Solutions

❌ Error: "Could not find table 'public.otp_codes'"

Solution: Run the missing table creation SQL:

CREATE TABLE IF NOT EXISTS public.otp_codes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT NOT NULL,
  code TEXT NOT NULL,
  purpose TEXT DEFAULT 'login',
  expires_at TIMESTAMPTZ NOT NULL,
  used BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_otp_email ON public.otp_codes(email);

❌ Error: "Email not sending"

Solutions:

  • Check email provider API key is correct
  • Verify sender email is verified in provider dashboard
  • Check spam/junk folder
  • For development, use the test_otp from response

❌ Error: "Invalid or expired OTP"

Solutions:

  • OTP expires after 10 minutes - request a new one
  • OTP can only be used once - get a new code
  • Check that email matches exactly (case-sensitive)

❌ Error: "JWT token expired"

Solution: Refresh token or re-authenticate:

// Re-authenticate
curl -X POST http://localhost:3000/api/auth/verify-otp \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "code": "123456"}'

❌ Error: "Port 3000 already in use"

Solution: Change port in .env:

PORT=3001
